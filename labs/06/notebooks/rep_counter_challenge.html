Challenge: Build a Rep Counter

You’ve learned how to detect body landmarks in real time. Now it’s your turn to build something of your own: a rep counter that watches a movement you choose and counts each repetition as you do it.

The Challenge

Pick a movement: a squat, a jumping jack, an arm raise, or something you invent - and write a program that:

  1. Watches your pose landmarks in real time
  2. Detects when you complete one repetition of the movement
  3. Counts and prints each rep as it happens

Sounds simple. There’s one bug almost everyone hits on the first try — let’s find it together before you start building.

Why Counting Is Trickier Than It Sounds

Here’s a tempting first attempt at counting arm raises:

if wrist_y < shoulder_y:
    count += 1

Run something like that and you won’t get one count per raise. You’ll get 30 counts per second for the entire time your arm is up, because the detector checks your pose about 30 times a second, and the condition stays True the whole time your arm is raised.

To count reps instead of frames, your program needs to remember what happened last time. Was the arm already up, or was it down? A rep should only count on the transition from down → up.

Worked Example: Arm Raise Counter

Here’s a complete rep counter for arm raises that solves the transition problem with a state variable. It also draws the live count directly on top of the video feed using canvas.get_context('2d') — the same 2D drawing surface draw_poses uses underneath. Anything drawn with ctx stays on screen until the next frame redraws it, so redrawing it every loop keeps the number in sync with your reps.

Run it, raise your arm a few times, and watch the count update live on the canvas. Study how state prevents the “30x per second” bug above — you’ll reuse this exact pattern for your own movement.

import cv
import graphics
import time

canvas = graphics.canvas()
camera = cv.start_camera(canvas)
detector = cv.start_pose_detector(camera)
ctx = canvas.get_context('2d')

state = "down"   # remembers whether the arm was already up last frame
count = 0

try:
  while True:
    poses = detector.get_detections()
    canvas.draw_poses(poses)

    if poses:
      pose = poses[0]
      wrist = pose[cv.POSE.LEFT_WRIST]
      shoulder = pose[cv.POSE.LEFT_SHOULDER]
      arm_up = wrist["y"] < shoulder["y"]

      if arm_up:
        count += 1

      # TODO: Replace the above if statement with this one...
      
      # if arm_up and state == "down":
      #   count += 1
      #   state = "up"
      # elif not arm_up and state == "up":
      #   state = "down"

    # draw the live count on top of the video feed
    ctx.fill_style = "rgba(0, 0, 0, 0.5)"
    ctx.fill_rect(10, 10, 150, 40)
    ctx.fill_style = "#ffffff"
    ctx.font = "28px sans-serif"
    ctx.fill_text(f"Reps: {count}", 20, 38)

    time.sleep(0.033)
finally:
  detector.stop()
  camera.stop()
  print(f"Final count: {count}")

Your Challenge

  • Which landmarks tell you about your movement? (e.g. LEFT_HIP and LEFT_KNEE for a squat)
  • What condition defines the “down” position vs. the “up” position?

Use the reference table below if you need a refresher on available landmarks.

Body part Constant
Shoulders cv.POSE.LEFT_SHOULDER, cv.POSE.RIGHT_SHOULDER
Elbows cv.POSE.LEFT_ELBOW, cv.POSE.RIGHT_ELBOW

Hips cv.POSE.LEFT_HIP, cv.POSE.RIGHT_HIP
Knees cv.POSE.LEFT_KNEE, cv.POSE.RIGHT_KNEE
